Structured data
Organised in fixed rows and columns with a predefined schema. Every record follows the same format. Think: anything that fits neatly in a spreadsheet or relational database.
+ Easy to query (SQL) + Easy to analyse - Rigid schema - ~20% of all data

Semi-structured data
Has some organisational markers (tags, keys, nesting) but no rigid tabular schema. Records can have different fields, nested objects, or varying depths. It sits between the two extremes.
+ Flexible schema + Machine-readable - Harder to query - Inconsistent structure
JSON

Unstructured data
No predefined format or schema at all. Cannot be stored in rows and columns without significant transformation. This is the vast majority of data generated in the world — and the hardest to analyse at scale.
+ Rich information + ~80% of all data - Needs AI/NLP to process - Not directly queryable
